1.1. 接口列表

接口名称 交互类型 发起方 接口类型 描述
订单回调 C01 平台 callback 订单发起核保后，平台将核保结果回调合作方
出单回调 C02 平台 callback 客户投保完成，保险公司生成保单并回传给平台后，平台回调合作方
续期结果通知 C05 平台 callback 保单进入到续期阶段，当保单的续期状态发生变化时，将通过此接口将最新的续期状态通知合作方
续保成功通知 C10 平台 callback 在保险公司有保单续保成功后，将通过此接口将续保出单信息通知合作方
回访状态通知 C11 平台 callback 当保单的回访状态发生变化时，将通过此接口将最新的回访状态通知合作方
退保状态通知 C12 平台 callback 当保单的发生退保时，将通过此接口将最新的保单状态通知合作方
人核状态变更通知 C13 平台 callback 当订单进入到人工核保流程中时，当有补充资料或者下发核保结论时，将通过次节扣通知合作方，并可以通过接口返回链接查看人工核保进度详情
订单关闭时间变更通知 C16 平台 callback 不同的订单状态会有不同的订单关闭时间，如果订单关闭时间发生变化将通过此通知合作方最新的订单关闭时间
推广费变更通知 C17 平台 callback 当保单的推广费产生变更时，会通过该接口通知给商户，请注意！由于某些特殊业务场景存在已生成推广费变更的情况，我们可能会对已推送的推广费数据做更新推送，请考虑这类数据的接收方案。
回执状态通知 C18 平台 callback 当保单回执状态发生变化是，通过此接口将最新的回访状态通知合作方
获取电子保单链接 R22 合作方 query 合作方根据保单号等信息在电子保单生成后可调用此接口获取电子保单链接
获取继续支付链接 R23 合作方 query 当订单进入待支付状态时，可通过此接口获取到继续支付链接，让用户继续发起支付
获取产品列表 R24 合作方 query 可通过该接口查询到产品的条款、责任信息并且能够实时获取产品的投保链接
保单状态查询 R25 合作方 query 可通过该接口查询到某一保单当前在平台的保单状态
保单列表查询 R26 合作方 query 可通过该接口查询到保单清单，作为每日对账依据
保单详情查询 R27 合作方 query 可通过该接口查询到某一条保单的所有详细信息，可作为保单同步的补偿机制接口
到期保单查询 R28 合作方 query 可通过该接口查询即将到期的保单清单， 并能够获取到支持连续投保产品的续保链接
订单列表查询 R29 合作方 query 可通过该接口查询订单清单
回执数据查询 R30 合作方 query 可通过该接口查询回执清单
回访数据查询 R31 合作方 query 可通过该接口查询回访清单
续期查询 R32 合作方 query 可通过该接口查询续期清单
订单详情查询 R33 合作方 query 可通过该接口查询订单详情
