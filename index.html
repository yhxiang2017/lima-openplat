1.1. 文档目的

本文档目的是平台为了统一与各合作方接口的交互协议、报文格式，方便合作方CPS对接后的数据回传。

1.2. 接口概述

1.2.1. 交互模式

属性 规则
使用协议 HTTPS/HTTP 请求响应模式
数据格式 JSON
传参方式 POST
MD5key 平台和每家合作方约定的一个唯一标识(私钥)
响应模式 支持同步与异步
POST参数 JSON报文的字符串进行Base64编码后生成的字符串
JSON 格式 报文格式统一 (注：请求时请求头中的 Content-Type 需设置为 application/json)

1.2.2. 数据类型

类型 描述
字符串 string 文本信息，标明最大的长度，该长度是按单字节进行计算数值Num数值信息
数字 number 数字类型，包含整数、浮点数、小数
时间 date 泛指日期（Date）和时间（Time），根据使用的情况可以仅包含日期部分，也可能包含小时、分钟等部分；
格式如： yyyy-MMddHH:mm:ss
时间皆为北京时间
布尔 boolean 用false代表假或否；用true代表真或是
枚举 enum 用于限制元素的取值，参考“字典”
金钱 所有金钱数据以分为单位

1.2.3. 回传/查询统一请求/响应模板

callback类型接口和query类型接口的请求/响应参数

{
    "sign": "MD5码值",
    "requestType": "C01/R27",
    "channelId": "分配给合作方的渠道编码",
    "data": "base64编码",
    "UUID": "cff6105b-081b-466e-aee8-7aac06f9cfb5"
}
参数名 类型 是否必传 参数说明
sign String true md5码，sign=MD5(data + signKey)
signKey从开放平台web端获取
requestType String true 接口唯一标识
channelId String true 分配给合作方的渠道编码
data String true 对请求/响应的报文进行base64编码，即：data = base64(报文)
uuid String true 能够唯一标识一次交互，返回报文中需要携带。主要用于双方调试跟踪，不进行任何保存和验证，仅在日志中打印出来

模板中的data的结构

数据文件采用标准的JSON格式，字符编码为UTF-8；JSON头（messageHead）和JSON体（request 请求、response响应）两部分组成。

{
  "messageHead":{},
  "request/response":{}
}

合作方发起请求报文

字段 字段名 类型 必传 备注
version 版本号 字符串 ME 匹配接口版本
requestType 交易类型 枚举 ME 见2.4
UUID 唯一标识 字符串 ME 能够唯一标识一次交互，返回报文中需要携带。主要用于双方调试跟踪，不进行任何保存和验证，仅在日志中打印出来
channelId 合作方编码 枚举 ME 由平台提供
sendTime 发送时间 时间 M yyyy-MM-dd HH:mm:ss，发送时取系统当前时间
asyn 是否异步接口 布尔 CE 缺省为同步
returnUrl 回调Url 字符串 C 对于异步返回接口(核保和出单)，平台回调请求合作方的地址
示例
{
    "messageHead":
    {
        "version": "1.0",
        "requestType": "C01",
        "UUID": "cff6105b-081b-466e-aee8- 7aac06f9cfb5",
        "channelId": "95851445214441411",
        "sendTime": "2012-01-01 11:20:00",
        "asyn": "ture",
        "returnUrl": "https://www.xxxx.xxx/xxx"
    },
    "request":
    {}
}

平台发起请求报文

字段 字段名 类型 必传 备注
version 版本号 字符串 ME 匹配接口版本
requestType 交易类型 枚举 ME 见2.4
UUID 唯一标识 字符串 ME 能够唯一标识一次交互，返回报文中需要携带。主要用于双方调试跟踪，不进行任何保存和验证，仅在日志中打印出来
channelId 合作方编码 枚举 ME 由平台提供
sendTime 发送时间 时间 M yyyy-MM-dd HH:mm:ss，发送时取系统当前时间
示例
{
    "messageHead":
    {
        "version": "1.0",
        "requestType": "C01",
        "UUID": "cff6105b-081b-466e-aee8-7aac06f9cfb5",
        "channelId": "109",
        "sendTime": "2012-01-01 11:20:00"

    "request":
    {}

请求返回报文

字段 字段名 类型 必传 备注
version 版本号 字符串 ME 匹配接口版本
requestType 交易类型 枚举 ME 见2.4
UUID 唯一标识 字符串 ME 能够唯一标识一次交互，返回报文中需要携带。主要用于双方调试跟踪，不进行任何保存和验证，仅在日志中打印出来
channelId 合作方编码 枚举 ME 由平台提供
sendTime 发送时间 时间 M yyyy-MM-dd HH:mm:ss，发送时取系统当前时间
messageCode 信息返回编号 String M 0为成功，其与都是失败
messageDesc 错误信息描述 String M
返回成功时的报文示例
{
    "messageHead":
    {
        "version": "1.0",
        "requestType": "C01",
        "UUID": "cff6105b-081b-466e-aee8- 7aac06f9cfb5",
        "channelId": "109",
        "sendTime": "2012-01-01 11:20:00",
        "messageCode": "0",
        "messageDesc": " 成功"
    },
    "response":
    {}
}
返回失败时的报文示例
{
    "messageHead":
    {
        "version": "1.0",
        "requestType": "C01",
        "UUID": "cff6105b-081b-466e-aee8- 7aac06f9cfb5",
        "channelId": "109",
        "sendTime": "2012-01-01 11:20:00",
        "messageCode": "SYSTEM000001E",
        "messageDesc": "JSON 报文格式错误"
    },
    "response":
    {}
}

1.2.4. 报文参数要求

  • M：必须出现；
  • C：条件出现；
  • ME：必须出现且于应答与请求保持一致
  • CE：如果出现，应答与请求保持一致
  • O：可选出现，参数名及值都可以不出现
